O que é clustering? Um algoritmo de agrupamento analisa vários pontos de dados e encontra automaticamente pontos de dados relacionados ou semelhantes entre si. Vamos dar uma olhada no que isso significa. Deixe-me comparar o agrupamento, que é um algoritmo de aprendizado não supervisionado, com o que você viu anteriormente com o aprendizado supervisionado para classificação binária. Dado um conjunto de dados como esse com os recursos x_1 e x_2. Com o aprendizado supervisionado, tínhamos um conjunto de treinamento com os recursos de entrada x e os rótulos y. Poderíamos traçar um conjunto de dados como esse e ajustar, digamos, um algoritmo de regressão logística ou uma rede neural para aprender um limite de decisão como esse. No aprendizado supervisionado, o conjunto de dados incluía tanto as entradas x quanto as saídas alvo y. Em contraste, no aprendizado não supervisionado, você recebe um conjunto de dados como esse com apenas x, mas não os rótulos ou rótulos de destino y. É por isso que quando eu ploto um conjunto de dados, ele fica assim, com apenas pontos em vez de duas classes indicadas pelos x e os o's. Como não temos rótulos alvo y, não podemos dizer ao algoritmo qual é a “resposta certa, y” que queríamos prever. Em vez disso, vamos pedir ao algoritmo que encontre algo interessante sobre os dados, ou seja, encontre alguma estrutura interessante sobre esses dados. Mas o primeiro algoritmo de aprendizado não supervisionado que você conhece é chamado de algoritmo de agrupamento, que procura um tipo específico de estrutura nos dados. Ou seja, observe o conjunto de dados dessa forma e tente ver se ele pode ser agrupado em clusters, ou seja, grupos de pontos semelhantes entre si. Um algoritmo de agrupamento, nesse caso, pode descobrir que esse conjunto de dados é composto por dados de dois clusters mostrados aqui. Aqui estão algumas aplicações do clustering.
