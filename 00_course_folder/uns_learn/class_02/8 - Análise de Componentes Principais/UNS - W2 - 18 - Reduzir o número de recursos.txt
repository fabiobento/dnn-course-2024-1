Esse é um algoritmo comumente usado para visualização. Especificamente, se você tiver um conjunto de dados com muitos recursos, digamos, 10 recursos ou até mesmo milhares de recursos, não é possível plotar dados de 1.000 dimensões. O PCA, ou análise de componentes principais, é um algoritmo que permite obter dados com muitos recursos, 50, 1.000, até mais, e reduzir o número de recursos para dois recursos, talvez três recursos, para que você possa plotá-los e visualizá-los. É comumente usado por cientistas de dados para visualizar os dados e descobrir o que pode estar acontecendo. Vamos dar uma olhada em como funciona a PCA, análise de componentes principais. Para descrever o PCA, vou usar como exemplo contínuo, se você tiver dados de uma coleção de carros de passageiros, e os carros de passageiros podem ter muitos recursos. Você pode saber o comprimento do carro ou a largura do carro, talvez o diâmetro da roda ou talvez a altura do carro e muitas outras características dos carros. Se você quiser reduzir o número de recursos para poder visualizá-los, como você pode usar o PCA para fazer isso? Para o primeiro exemplo, digamos que você receba um conjunto de dados com dois recursos. O recurso x_1 é o comprimento do carro, assim como, e o segundo recurso x_2, é a largura do carro, que é medida assim. Acontece que na maioria dos países, devido às restrições quanto à largura da estrada em que os carros circulam, a largura do carro, que deve caber na largura da estrada de uma única faixa, tende a não variar muito. Por exemplo, nos Estados Unidos, a maioria dos carros tem, digamos, cerca de 1,8 metros de largura, ou seja, pouco menos de seis pés. Se você tivesse uma coleção de carros e o conjunto de dados de comprimento e largura dos carros, descobriria que o conjunto de dados poderia ter a seguinte aparência, onde x_1 varia um pouco porque alguns carros são muito longos e x_2 varia relativamente pouco. Se você quiser reduzir o número de recursos, bem, uma coisa que você pode fazer é usar x_1 porque x_2 varia relativamente pouco de carro para carro. Acontece que o PCA é um algoritmo que, quando aplicado a esse conjunto de dados, decide mais ou menos automaticamente usar x_1, mas pode fazer muito mais do que isso. Aqui está um exemplo mais complexo. Digamos que o recurso x_1 seja o comprimento do carro, então isso varia um pouco, e o recurso x_2 aqui é a altura do carro, que também varia um pouco. Você não quer escolher apenas x_1, o comprimento, ignorar x_2 a altura e também não quer escolher apenas x_2, a altura, e ignorar x_1, o comprimento. Parece que x_1 e x_2 têm informações úteis. Mas por que temos o eixo z que corresponde a algo do tamanho do carro? Dado um carro como este aqui, sua coordenada, ou seja, o valor no eixo x nos diz o comprimento do carro e a coordenada é justa, qual é essa distância? Da mesma forma, sua coordenada, ou seja, qual é essa distância no eixo x_2, nos diz qual é a altura do carro. Se agora vamos usar o eixo z como um recurso para capturar o que sabemos sobre esse carro, então é a coordenada no eixo z, ou seja, essa distância. Isso nos diz aproximadamente qual é o tamanho do carro. Vamos formalizar isso nos próximos vídeos. Mas a ideia do PCA é encontrar um ou mais eixos novos, como z, para que, ao medir suas coordenadas de dados no novo eixo, você ainda tenha informações muito úteis sobre o carro.
